## 3. New blockers discovered during Pass 2 (not in Pass 1's analysis)

### 3.1 Doom Queen e3 silent failure when both P-zones occupied

**Diagnostic** (attempts 4, 5): With Oblivion + Kepler scales, Sky King ignition destroys Doom Queen on M1. Doom Queen e3 should fire ("If destroyed in MZONE: place in P-zone"). The engine does NOT surface SELECT_PLACE for e3 — it appears silently auto-NO'd because no available P-zone exists. This is a non-obvious failure mode worth documenting in the Pass 1 audit's chainTargets coverage table.

### 3.2 Gilgamesh material list excludes Copernicus when Lance picked first (link-arrow constraint)

**Diagnostic** (attempt 7): When Gilgamesh is placed at EMZ_R (auto SELECT_PLACE) and Lance is selected as first material, Copernicus is **NOT in the SELECT_UNSELECT_CARD options** for the second material. Likely a Link-arrow MZONE-position constraint. This forces Pass 2 back to Doom Queen + Copernicus as Gilgamesh materials (not Lance + Copernicus to preserve Doom Queen on field for the Sky King ignition target — that would be the alternate path).

### 3.3 Pendulum-Summon entry-point card-name is the LOWER-scale slot, not arbitrary

**Diagnostic** (attempt 2): I tried `pendulum-summon: cardName="D/D Gryphon"` (S5 scale) — failed with divergence. Engine surfaces pendulum-summon under the **Kepler** name (S1 scale, the LOWER physical zone index, regardless of scale value). Trivial mistake but not documented.

### 3.4 Solomon's "D/D card" search is archetype-restricted (0xaf), excludes Dark Contract (0xae)

**Diagnostic** (research not in-engine): Solomon detach searches "D/D card from Deck". Per cards.cdb:
- Headhunt: setcode 0x10af → matches 0xaf ✓ (search succeeds)
- Eternal Darkness: setcode 0xae → does NOT match 0xaf ✗ (search fails)
- Zero King: setcode 0xaf00ae → matches 0xaf ✓ (would succeed)

So Solomon CAN'T search Eternal Darkness despite "Dark Contract" pattern. Need Caesar R4/R6 specifically (text "Dark Contract from Deck" matches 0xae setcode).

This blocks the +1-Eternal-Darkness extension because Caesar R4 GY-send is unreachable.
