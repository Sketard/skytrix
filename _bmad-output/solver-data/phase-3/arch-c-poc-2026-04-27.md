## Architecture C summary

LLM annotates trajectory states **offline** to produce labeled (state, ranked_actions) corpus.
Trained policy network distills LLM signal — runtime has zero LLM calls. Vision-aligned
because the LLM is teacher only; the deployed policy network is deck-agnostic.

Critical discovery this session: **subagent orchestration via Claude Code's `Agent` tool**
provides LLM annotation labor with **zero API cost** — uses the user's existing Claude Code
subscription quota. Eliminates the $50-100 budget Architecture C originally required.

## Findings

### Robust signals
- **Top-3 holds across all conditions** at 60-80%
- **Top-5 = 100% across 20 prompts** — ground truth is ALWAYS within top-5 LLM ranks
- **New-fixture radiant 60/80% top-1/top-3** — comparable to baseline → not fixture-specific fluke
- **Subagent orchestration validated** — 10-20 parallel agents return parseable JSON reliably

### Scaffolding sensitivity
- Without expectedBoard hint: top-1 collapses 70% → 0%
- Top-3 less affected: 80% → 60%
- LLM clearly USES the expectedBoard reference to calibrate top-1 picks
- Implication: for vision-aligned distillation, must train policy WITHOUT exposing the hint

### Misses are informative, not hallucinatory
- Branded step 0 with hint: LLM picked Branded Fusion (canonical YGO play); ground truth =
  NS Fallen of the White Dragon (the authored trajectory's specific Cartesia → Secreterion
  pivot, intentionally non-standard)
- All other misses follow this pattern: LLM picks reasonable alternative; ground truth is
  in top-3 or top-5

This is GOOD signal — the LLM understands YGO mechanics; disagreement is over branch
choice, not fundamental confusion.

## Phase 1 strategy — hybrid distillation

| Approach | Top-3 expected | Vision-aligned? |
|---|---:|:-:|
| Annotate WITH hint, deploy WITH hint | 80% | ✗ (scaffolding at runtime) |
| Annotate WITHOUT hint, deploy WITHOUT hint | 60% | ✓ but signal weaker |
| **Hybrid: annotate WITH hint, distill policy WITHOUT hint** | 80% (training) | **✓ (clean distillation)** |

The hybrid uses expectedBoard as **privileged supervisor information** at annotation time.
The student policy network sees only state features at runtime — it never observes the hint.
This is standard knowledge distillation with privileged information (Vapnik 2009).
