## Promotion mechanism

Manual file copy, mirroring graph-ml-v1 convention:

```bash
cp data/trained-weights/neural-mlpv3-gate-seed7.json \
   data/trained-weights/neural-tier-a-latest.json
```

The historical artefact `neural-mlpv3-gate-seed7.json` is preserved alongside
the canonical promotion (no rename, no delete) so the promotion is auditable
post-hoc.

There is no `neural-tier-a-latest.pre-X-backup.json` (no previous canonical
neural weights existed). Future promotions should follow the graph-ml-v1
backup convention (`tier-a-latest.pre-vN-backup.json`).

## Wiring change in solver-worker.ts

```typescript
// Phase B (graph-ml-v2 / neural) and graph-ml-v1 are mutually exclusive —
// when both env vars are set, neural wins (Phase B design doc §2; mirrors
// evaluate-structural.ts). NeuralFeatureRanker wraps RouteAwareRanker and
// adds a neural bonus over the deck-agnostic feature vector.
const neuralWeights = loadNeuralWeightsIfEnabled({ dataDir });
const tunedWeights = neuralWeights ? undefined : loadTunedWeightsIfEnabled({ dataDir });
let ranker: ActionRanker;
let neuralRanker: NeuralFeatureRanker | undefined;
if (neuralWeights) {
  const nr = new NeuralFeatureRanker(routeAwareRanker);
  nr.setInterruptionTags(allConfigs.interruptionTags);
  nr.setInterruptionWeights(allConfigs.interruptionWeights);
  nr.setNeuralWeights(neuralWeights);
  neuralRanker = nr;
  ranker = nr;
} else if (tunedWeights) {
  // ... unchanged graph-ml-v1 path
}
```

Per-solve setup added in `runSolve()` after `scorer.setCardMetadata()`:

```typescript
if (neuralRanker) {
  neuralRanker.setMetadata(cardMetadata);
  neuralRanker.setMainDeck(task.duelConfig.mainDeck);
  neuralRanker.setExtraDeck(task.duelConfig.extraDeck);
}
```

`routeAwareRanker.setArchetypeExpertise(filteredExpertise)` already runs
per-solve unchanged — neural wraps RouteAwareRanker at the outer layer, so
expertise routing is preserved (just with an additional neural bonus on top
when env-enabled).

## How to use

### Default (untouched, no env vars)
- Both env vars off → behaviour byte-identical to pre-ship: solver uses
  `routeAwareRanker` plain, no neural, no graph weighting.

### Phase B v2 (canonical, neural opt-in)
```bash
SOLVER_USE_NEURAL_WEIGHTS=1 [your normal command]
```
Loader picks up `neural-tier-a-latest.json` automatically. Override file
with `SOLVER_NEURAL_WEIGHTS_FILE=<basename>` if needed.

### Graph-ml-v1 (legacy, still wired)
```bash
SOLVER_USE_TUNED_WEIGHTS=1 [your normal command]
```
Untouched. Loads `tier-a-latest.json`.

### Both set
Neural wins. graph-ml-v1 loader silently disabled.
