P0-1 — Player Markers on Chain Badges

skytrix has two distinct chain badge components, each with its own visual language: the board badge (pulsing circle on cards in zones, the one Axel meant) and the chain overlay badge (Master Duel-style italic numeral on stacked cards during chain build). The recommendation is to split each by player while preserving its existing aesthetic and animation. Palette: blue (self) / amber (opponent), colorblind-safe.

Design tokens — board badge (① pulsing circle)

Self bg
rgba(30, 90, 200, 0.9) — same blue as today's badge. Continuity preserved.
Self ring
rgba(96, 165, 250, 0.95) — lighter blue (blue-400). Ties the ring to the fill semantically (replaces the current neutral grey ring).
Opp bg
rgba(180, 83, 9, 0.92) — amber-700. Saturated enough to hold contrast against bright card art. Colorblind-safe pairing with the blue.
Opp ring
rgba(245, 158, 11, 0.95) — amber-500. Same fill→ring lightness relationship as self.
Animation
Existing chain-badge-pulse keyframes preserved — opacity 0.4→1, scale 0.9→1.1, 1s ease-in-out infinite alternate. Untouched.
Negated state
Identity color suppressed. Background rgba(70, 70, 70, 0.85), border rgba(160, 160, 160, 0.8), animation removed. Negation overrides identity.

Design tokens — chain overlay badge (② italic numeral)

Self stroke
#4a90d9 — same cyan as today's overlay badge. Continuity preserved.
Opponent stroke
#f59e0b — amber-500. Hue distance from cyan = ~190°. Colorblind-safe (deuteranopia/protanopia/tritanopia simulators all keep the pair distinguishable).
Fill
#ffffff — both. White-on-stroke remains the readable Master Duel pattern.
Glow
Self rgba(74, 144, 217, 0.85) — Opp rgba(245, 158, 11, 0.85). Two-layer text-shadow as today.
Resolution pulse (orthogonal)
Existing gold token rgba(255, 215, 0, 0.6) for chain-resolve-glow keyframes. Identity-agnostic. Preserved as-is.
Negated state
Existing greyscale + grey shake takes over (overrides player color). No conflict.
A11y
LiveAnnouncer wording change: "You added Chain Link N: CardName" / "Opponent added Chain Link N: CardName".

Data already there
ChainLinkState.player + buildHandChainBadges() / buildOpponentHandChainData() (chain-badge.utils.ts) already split links by ownership. The grunt work is done.

Add chain-badge--opponent modifier in pvp-board-container.component.scss (around line 532). Branch background + border on the modifier. Pass an isOpponent boolean from the template based on link ownership.
Component change — chain overlay badge (②)
Add .chain-badge--opponent modifier in pvp-chain-overlay.component.scss (around line 282). Branch -webkit-text-stroke + text-shadow on the modifier. Use the existing link.player against ownPlayerIndex.
Component change — hand row badges
Same modifier pattern in pvp-hand-row.component.scss. Note the hand row badges follow the board badge style (pulsing circle), not the overlay style.
Surfaces touched
Three SCSS files, three template branches. No TS logic change.
Animations
NO new animation introduced. chain-badge-pulse (board) and chain-resolve-glow (overlay) preserved exactly. Only color tokens swap.
Reduced motion
Inherits existing prefers-reduced-motion rules. No new exceptions needed.
A11y wording change
