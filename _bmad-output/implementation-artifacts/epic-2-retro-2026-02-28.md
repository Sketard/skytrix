## Successes & Strengths

1. **Signal-based `roomState` state machine** — Started with 6 states in Story 2-1, incrementally enriched with RPS transitions (2-3) and `duel-loading` (2-4). Zero destructive refactoring. Foundation for the entire epic.

2. **Previous Story Intelligence** — Each story's Dev Notes explicitly documented patterns and anti-patterns from prior stories. The polling `catchError(() => EMPTY)` pattern from Story 2-1 C1 fix was correctly applied in all subsequent stories — zero recurrence.

3. **Angular dev checklist effectiveness** — The `[class]` binding bug that recurred in 3/6 Epic 1 stories had 0 occurrences in Epic 2. Touch target violations dropped from 3/6 to 0/4. Findings per story reduced from 13 (Epic 1) to 8 (Epic 2) — **38% improvement**.

4. **Backend contract fixes inline** — Story 2-2 discovered 4 mismatches between planned API and actual implementation (`{id}` vs `{roomCode}`, missing pessimistic lock, missing limit, HTTP status codes). All 4 fixed within the same story rather than deferring.

5. **3-tap flow achieved** — Deck builder → lobby → join → RPS → duel in 3 taps. UX target met. Deep link with `returnUrl` auth redirect enables seamless sharing.

6. **Quality trend positive** — 32 findings across 4 stories (avg 8/story vs 13 in Epic 1). Only 2 Critical findings (vs 10+ in Epic 1). All findings fixed pre-merge. Zero production incidents.

## Challenges & Growth Areas

1. **`prefers-reduced-motion` still missed (2/4 stories)** — Flagged in Story 2-1 (M2: spinner internals) and Story 2-4 (H1: duel-loading spinner). Only recurring item from the Angular checklist. Root cause: never tested with `prefers-reduced-motion` enabled in dev.

2. **Multi-player bugs invisible in solo dev** — Story 2-3 C1: RPS perspective bug where joiner saw inverted choices and wrong winner text. Only detectable with 2-player testing. Code review AI caught it, but the pattern highlights a testing blind spot.

3. **Backend/story desynchronization** — Story 2-2 found 4 contract mismatches. Root cause: backend was implemented before detailed stories were created. Planning artifacts and actual code diverged on details (`{id}` vs `{roomCode}`, limit, lock strategy).
