Architecture — comment ajouter un 4ème thème

Le système est purement déclaratif. Pour ajouter un thème "Volcano" : 

.board-host[data-theme="volcano"] {
  --duel-bg-base: #1a0a05;
  --duel-bg-radial: radial-gradient(ellipse at 50% 80%, rgba(255, 80, 30, 0.3) 0%, transparent 60%);
  --duel-vignette: rgba(40, 5, 0, 0.8);
  --duel-accent-primary: #ff6b35;
  --duel-accent-soft: rgba(255, 107, 53, 0.18);
  --duel-zone-border: rgba(255, 107, 53, 0.25);
  --duel-zone-bg: rgba(40, 8, 0, 0.45);
  --duel-emz-border: rgba(255, 140, 50, 0.55);
  --duel-emz-glow: rgba(255, 107, 53, 0.35);
  --duel-emz-bg: rgba(180, 50, 0, 0.18);
  --duel-pile-tint: rgba(255, 107, 53, 0.12);
  /* ...plus animation idle "magma-flow" si besoin */
}

Aucun HTML modifié. Aucun composant Angular impacté. La règle Animation Parity tient (orchestrator → tokens via host class). Replay hérite gratuitement : il suffit que la classe data-theme soit appliquée par le composant racine PvP ET Replay.

Implémentation Angular : un ThemeService avec un signal currentTheme: Signal<'classic' | 'cosmic' | 'forest'> bindé en attribut sur le host. Sélection persistée par utilisateur (LocalStorage). Optionnel : par-deck (le thème voyage avec le deck choisi).
